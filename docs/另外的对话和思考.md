## 核心架构思路

### 1. 世界规则引擎 (World Rules Engine)

```typescript
// 世界规则分层模型
interface WorldRules {
  // 物理法则层 - 不可违背的基础规则
  physics: PhysicsRules;
  
  // 社会规则层 - 可演变的社会结构
  society: SocietyRules;
  
  // 叙事规则层 - 故事发展的约束与引导
  narrative: NarrativeRules;
  
  // 用户自定义层 - 创作者设定的特殊规则
  custom: CustomRules;
}
```

### 2. AI认知模型 (AI World Cognition)

不是让AI"学习"规则，而是让AI"理解"规则的**语义意图**：

- **规则解释器**：将规则转化为AI可理解的"行为约束"
- **上下文注入**：在每次生成前，将相关规则作为上下文注入
- **一致性校验**：生成后验证是否符合规则

### 3. 模块化设计

### 4. 关键优化点

**规则表达优化**：
- 用"约束 + 示例"代替纯文本描述
- 规则优先级系统（硬约束 vs 软引导）
- 规则冲突检测与解决

**AI理解优化**：
- 规则向量化，支持语义检索
- 动态规则组装（根据当前场景选择相关规则）
- 规则影响评估（预测某规则对生成内容的影响）

**创作者体验优化**：
- 交互式规则构建向导
- 规则效果预览（"如果设定这条规则，会发生什么"）
- 规则模板市场

## SillyTavern 核心机制总结

### 1. Character Card
- 一个"角色定义包"，包含人设、性格、背景等
- 本质是**静态 prompt 模板**

### 2. World Info / Lorebook（最值得关注）
```
Entry 结构：
├── Keywords (主触发词)
├── Secondary Keywords (次触发词，支持 AND/OR/NOT 逻辑)
├── Content (要插入的 lore 内容)
├── Insertion Order (插入位置，数字越小越靠前)
├── Priority (优先级，token 超限时决定谁被保留)
├── Constant (是否常驻)
└── Probability (触发概率)
```

**核心逻辑**：
- 扫描用户输入 → 匹配关键词 → 将对应 Content 插入上下文
- **递归触发**：插入的 lore 中的关键词可以触发其他 entry
- **Token 预算管理**：优先级低的被丢弃

### 3. 上下文管理
- RAG 文档检索
- 自动摘要压缩历史
- Data Bank 知识库

---

## SillyTavern 的优点与局限

| 优点 | 局限 |
|------|------|
| 触发机制灵活（关键词+正则+逻辑） | 规则是"被动触发"，不是"主动理解" |
| Token 预算管理实用 | 缺乏规则间的关联性和一致性检查 |
| 递归触发支持复杂 lore | 无规则优先级/冲突解决机制 |
| 社区生态成熟 | 纯文本规则，难以表达复杂约束 |

## 我们的优化方向

基于 SillyTavern 的经验，我建议我们做以下**关键差异化**：

### 1. 从"关键词触发"升级为"语义理解"
```typescript
// SillyTavern 方式：关键词匹配
keywords: ["魔法", "法术"] → 触发魔法系统规则

// 我们的方式：语义意图匹配
intent: "当场景涉及超自然力量使用时" → 自动应用魔法规则
```

### 2. 从"扁平 Entry"升级为"规则图谱"
```typescript
// 规则间显式关联，支持继承、冲突、依赖
interface RuleNode {
  id: string;
  content: string;
  parentRules?: string[];      // 继承
  conflictsWith?: string[];    // 冲突检测
  dependsOn?: string[];        // 依赖
  overrides?: string[];        // 覆盖
}
```

### 3. 从"静态插入"升级为"动态组装"
```typescript
// 根据当前场景智能选择、组合、压缩规则
interface RuleAssembly {
  select: (context: SceneContext) => RelevantRules;
  compose: (rules: RelevantRules) => OptimizedPrompt;
  validate: (output: string, rules: ActiveRules) => boolean;
}
```

### 4. 新增：规则效果预测
让用户在设定规则时，能预览"这条规则会如何影响 AI 的生成"。
